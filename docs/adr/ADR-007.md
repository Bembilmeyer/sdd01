# ADR-007: Философия и уровни тестирования

- **Статус:** принято
- **Дата:** 2026-06-30
- **Владелец решения:** заказчик
- **Связь с VISION:** D1-F1, D1-F2

## Контекст

Нужно явно выбрать философию тестирования, потому что в продукте сочетаются deterministic-контуры и AI/RAG-поведение с высокой степенью неопределённости. Без явного решения команда по умолчанию может скатиться в неподходящий для AI-контуров Test-First.

## Рассматриваемые варианты

| Вариант | Trade-off | При каком условии выигрывает |
|---------|-----------|------------------------------|
| Experiment-first | Быстрее исследует качество поиска и генерации, но требует дисциплины, чтобы вовремя зафиксировать результат регресс-тестами | Если в продукте много неопределённости вокруг retrieval и LLM-ответов |
| Test-first / TDD | Даёт строгую инженерную дисциплину и ранние контракты, но плохо подходит для ещё не стабилизированного AI-поведения | Если поведение системы заранее хорошо определено и слабо зависит от эвристик/моделей |

## Решение

Заказчик выбрал `Experiment-first`.

## Обоснование

Для D1-F1 критично сначала найти рабочее поведение поиска и генерации ответа, а уже потом фиксировать его автотестами. При этом deterministic-части системы не должны выпадать из инженерной дисциплины, поэтому для них сохраняются обязательные автоматические тесты.

## Отвергнутые варианты и почему

> Самая ценная часть ADR — не теряйте её.

- **Test-first / TDD** — отвергнут, потому что для нестабилизированного AI/RAG-контура создаёт ложную определённость и замедляет исследование качества решения.

## Последствия

Приняты уровни тестирования:
- для `auth`, `API contracts`, `session persistence`, `source connectors`, `authorization` — автоматические unit/integration tests обязательны;
- для `retrieval`, `ranking`, `answer generation` — сначала прототипы, ручные сценарии и интеграционные проверки качества, затем закрепление регресс-тестами и evaluation-наборами;
- перед production-недрением нужен минимальный smoke-набор пользовательских сценариев для golden path.
