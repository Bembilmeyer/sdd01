# ADR-002: Backend-стек приложения

- **Статус:** принято
- **Дата:** 2026-06-30
- **Владелец решения:** заказчик
- **Связь с VISION:** D1-F1, D1-F2

## Контекст

После выбора модульного монолита нужно определить backend-стек, который будет нести API приложения, оркестрацию поиска и генерации ответа, интеграцию с корпоративной аутентификацией и работу с хранилищами.

## Рассматриваемые варианты

| Вариант | Trade-off | При каком условии выигрывает |
|---------|-----------|------------------------------|
| Python + FastAPI | Быстрая разработка AI/RAG-функций и богатая экосистема, но слабее корпоративная стандартизация | Если AI-контур важнее стандартов корпоративной платформы |
| C# / ASP.NET Core | Сильная типизация, зрелый enterprise-стек и соответствие внутренним стандартам, но AI-интеграции могут быть менее удобны, чем в Python | Если в компании .NET является стандартом и важна интеграция с существующим контуром |
| Java + Spring Boot | Зрелый enterprise-стек и операционная предсказуемость, но более тяжёлый старт и слабее соответствие текущему корпоративному стандарту | Если организация стандартизирована на Java |

## Решение

Заказчик выбрал `C# / ASP.NET Core`, поскольку корпоративным стандартом является .NET.

## Обоснование

Решение привязано к организационному ограничению, которое влияет на сопровождение, найм, эксплуатацию и интеграцию с внутренним контуром. Для D1-F1 и D1-F2 это снижает риски внедрения в корпоративной среде, даже если часть AI-функций потребует более внимательного выбора библиотек.

## Отвергнутые варианты и почему

> Самая ценная часть ADR — не теряйте её.

- **Python + FastAPI** — отвергнут, потому что противоречит корпоративному стандарту, несмотря на более удобную AI-экосистему.
- **Java + Spring Boot** — отвергнут, потому что не соответствует заявленному внутреннему стандарту и не даёт дополнительных преимуществ для текущего продукта.

## Последствия

Нужно дальше выбрать UI-стек, OLTP-хранилище, поисковый индекс и способ интеграции .NET-приложения с LLM/RAG-контуром. Стоит отдельно проверить зрелость библиотек для embedding, reranking и orchestration в выбранном .NET-стеке.
