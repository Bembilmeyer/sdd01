# VISION: Поиск по корпоративной базе знаний

> Документ заполняется по ходу интервью (навык `vision`). Незаполненное — это не пустота, а явный gap с владельцем и сроком: `[NEEDS CLARIFICATION: вопрос | владелец | когда решаем]`.

## Акторы

- **Заказчик / спонсор:** Руководитель службы технической поддержки
- **Основной пользователь:** Сотрудник службы поддержки
- **Вторичные пользователи:** Рядовые сотрудники компании, у которых иногда возникают вопросы о функциях информационных систем, с которыми они работают

## Намерение

- **Зачем:** Ускорить работу сотрудников за счёт быстрого нахождения по ключевым словам точной информации о возможностях информационных систем компании.
- **Почему сейчас:** Возможности информационных систем стали многочисленными, и пользователи перестали держать их в голове. Кроме того, постоянно появляются новые пользователи, время онбординга которых нужно сократить.

## Jobs-to-be-done (основной пользователь)

Быстро находить ответ на вопрос об информационных системах компании во внутренней базе знаний.

## Golden path

1. Сотруднику службы поддержки поступает запрос от клиента.
2. Сотрудник поддержки вводит в интерфейс приложения ключевые слова для поиска релевантных функций информационных систем, которые могут решить запрос клиента.
3. Приложение выдаёт топики, которые считает релевантными запросу сотрудника.
4. Сотрудник поддержки вводит уточняющие слова либо сообщает приложению, что поиск пошёл не в ту сторону.
5. Когда сотрудник удовлетворён ответом приложения, он закрывает сессию.
6. Перед закрытием приложение сохраняет данные сессии в БД для анализа в будущем.

## Что считаем успехом

Общее сокращение времени ожидания обращающихся в службу поддержки на получение ответа.

## Non-goals (что сознательно НЕ делаем)

- Не делаем мобильную версию приложения, только десктоп.
- Не делаем сложный UI: достаточно минимального набора инструментов для ввода поиска, чтения результатов, а также средств для логина и логаута сотрудника.
